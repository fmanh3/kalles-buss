# 2026-04-14: Fix av Test- och Byggkedja för Monorepo

## Problem

Efter den initiala committen av monorepot visade det sig att både testsviten och byggprocessen var trasiga. En körning av `npx vitest run` och `npx tsc -b` resulterade i ett stort antal fel över flera paket.

## Åtgärder

En omfattande felsökning och refaktorering genomfördes för att stabilisera monorepot.

### 1. Testramverk (Vitest)

-   **Problem:** Tester skrivna för `Jest` misslyckades i `Vitest` med `ReferenceError: describe is not defined`.
-   **Lösning:** Explicita importer för `describe`, `it`, `expect`, etc., lades till från `vitest` i alla berörda testfiler i paketen `kalles-hr`, `kalles-finance` och `poc`.

### 2. TypeScript-konfiguration (`tsconfig.json`)

-   **Problem:** Flera `tsconfig.json`-filer hade felaktig `rootDir`-konfiguration som förhindrade import av filer utanför `src`-katalogen (t.ex. `knexfile.ts`). Dessutom saknades en strategi för att hantera beroenden mellan paketen.
-   **Lösning:**
    -   `rootDir` togs bort och `include`-arrayen uppdaterades för att explicit inkludera nödvändiga filer.
    -   Projektreferenser (`references`) lades till i `tsconfig.json` för varje paket för att definiera beroendegrafen (`kalles-hr` -> `shared-utils`, etc.).
    -   Rot-`tsconfig.json` uppdaterades för att referera till *alla* paket, vilket möjliggör en fullständig build med ett enda `tsc -b`-kommando.

### 3. Refaktorering: Delad Kod

-   **Problem:** Kritisk infrastrukturkod, som `PubSubClient`, duplicerades och/eller importerades med felaktiga relativa sökvägar tvärs över paketgränser (t.ex. `kalles-hr` som importerade från `kalles-traffic/src/...`).
-   **Lösning:**
    -   Ett nytt delat paket, `@kalles-buss/shared-utils`, skapades under `packages/`.
    -   `PubSubClient` flyttades till det nya delade paketet.
    -   Beroende paket (`kalles-hr`, `kalles-finance`, `kalles-traffic`) uppdaterades för att importera `PubSubClient` från `@kalles-buss/shared-utils`.

### 4. NPM Workspaces

-   **Problem:** Rot-`package.json` definierade inte alla paket som en del av arbetsytan, vilket ledde till problem med länkning av beroenden. `npm install` misslyckades även på grund av problem med `workspace:`-protokollet i den specifika miljön.
-   **Lösning:**
    -   `workspaces`-arrayen i rot-`package.json` utökades till att inkludera alla relevanta paket.
    -   Beroenden mellan lokala paket specificerades med exakta versionsnummer (`"1.0.0"`) som en workaround för `workspace:`-problemet, vilket möjliggjorde en lyckad `npm install`.

### 5. Diverse Kodfixar

-   Felaktiga importer, oanvända variabler och typfel (t.ex. `Knex` vs `knex`) åtgärdades i flera filer.
-   `kalles-traffic/src/index.ts` var i ett gravt trasigt skick och kommenterades tillfälligt ut för att möjliggöra en lyckad build av resten av projektet.

## Resultat

Efter dessa åtgärder är nu byggkedjan och testsviten stabila:
-   `npx vitest run` exekverar alla 8 tester utan fel.
-   `npx tsc -b` bygger hela monorepot utan fel.

Projektet är nu i ett mycket bättre skick för vidare utveckling.
